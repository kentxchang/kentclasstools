🔥 Firebase 通用專案設定教學

你需要建立 兩個 專案：一個給通用工具，一個給「剛好測」

已包含 100 年免維護設定方案

第一部分：通用專案設定 (剛好學, 白板, 簡報)

1-3

建立專案、開啟匿名登入、建立資料庫

請依照 「建立專案」、「開啟匿名登入 (Authentication)」 和 「建立資料庫 (Firestore)」 的步驟操作。

專案名稱建議： Universal-Tools
通用專案規則
4

設定 100 年有效期的安全性規則 (通用工具)

這是給 剛好學、剛好白板、剛好簡報 專用的規則，可讓資料在 2125 年前都能讀寫，無需每年更改。 

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 匹配資料庫中的任何文件路徑
    match /{document=**} {
      // 允許讀取和寫入，條件是：當前時間小於 2125 年 1 月 1 日
      allow read, write: if request.time < timestamp.date(2125, 1, 1);
    }
  }
}

第二部分：剛好測專案設定 (必需單獨申請)

A

建立第二個專案（剛好測）

重要： 由於剛好測的資料（題庫、成績）需永久保存，我們必須將它與其他工具分開。

請重複步驟 1-3：「建立專案」、「開啟匿名登入」 和 「建立資料庫」。

專案名稱建議： Quiz-Database
剛好測專用規則
B

設定剛好測的安全性規則

這是專為 剛好測 的資料結構設計的規則，確保老師可管理題庫、學生可匿名作答與讀取成績。 

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 匹配資料庫中的任何文件路徑
    match /{document=**} {
      // 允許讀取和寫入，條件是：當前時間小於 2125 年 1 月 1 日
      // (所有資料都設為 100 年效期，無需擔心過期)
      allow read, write: if request.time < timestamp.date(2125, 1, 1);
    }
  }
}

提示： 雖然剛好測有特殊的資料結構，但由於我們讓所有使用者都能讀寫，所以可以直接沿用與通用專案相同的 100 年通用規則，設定方式完全相同。請同樣貼上 步驟 4 的代碼即可。

第三部分：取得設定碼與應用

5 (新)

在主程式中設定 Firebase (推薦新方法！)

全新功能：現在您可以直接在主程式中完成 Firebase 設定，無需下載或編輯檔案！

1.

開啟主程式並點擊按鈕

在瀏覽器中開啟「剛好系列主程式」，點擊首頁右上角找到 「Firebase 設定」 按鈕並點擊。

2.

貼上 Firebase 設定值

將您從 Firebase 複製的六行 Config 內容（例如通用專案或剛好測專案的設定）貼到文字框中。

apiKey: "AIzaSy...",
authDomain: "xxx.firebaseapp.com",
...
3.

套用設定

點擊 「套用設定」 按鈕。系統會自動解析並儲存您的 Firebase 設定，頁面會重新載入，設定會儲存在瀏覽器的 localStorage 中。

生成異機專用網址（選用但推薦）

重新點擊 「Firebase 設定」 按鈕，然後點擊 「生成異機專用網址(可免再設定firebase)」 按鈕。

  • 系統會生成含有您 Firebase 設定的網址，方便分享給學生或列印使用。
  • 其他裝置開啟後會自動載入您的 Firebase 設定。
注意事項： 設定會儲存在瀏覽器的 localStorage 中。生成的短網址和 QR Code 包含您的 Firebase 設定資訊，請妥善保管。
6 (舊)

使用 Firebase 設定工具（舊方法）

開啟剛好系列 Firebase設定工具.html，依照頁面指引貼上剛才取得的 firebaseConfig 並點選生成。

  • 工具會自動產生整合好設定的「剛好系列主程式」HTML 檔。
  • 下載後即可上傳靜態網頁空間使用，如 GitHub Pages。
  • **工具網址：** 點此開啟設定工具

第四部分：上傳您的主程式 (使用 EzPage)

當您完成 Firebase 設定並透過 設定工具 (步驟 6) 生成最終的 HTML 檔案後，下一步就是將其發佈到網路上，讓學生可以存取。我們推薦使用 「EzPage」 這個工具，它能讓您輕鬆地將靜態網頁上傳到 GitHub Pages，無需繁瑣的 Git 指令。

1

登入 EzPage 並授權

前往 EzPage 官方網站，點擊「立即開始」。點選「新增帳號」，並選擇「使用 Google 帳號」進行登錄。過程中請同意授權 EzPage 存取您的 GitHub 帳號。

2

上傳 HTML 檔案並部署

登入後，將您使用「firebase設定工具」生成的 HTML 檔案，直接拖曳到 EzPage 頁面右側的檔案上傳區。設定一個 「專案名稱」 (將成為網址的一部分)，然後按下 「部署 HTML」。

3

驗證與瀏覽網站

部署完成後，觀察專案列表，待 GitHub 倉庫的圖示變成綠色打勾時，表示網站已準備就緒。點擊 「查看網站」，即可開啟您剛上線的主程式！
