🔥 Firebase 通用專案設定教學

你需要建立 **兩個** 專案：一個給通用工具，一個給「剛好測」

已包含 100 年免維護設定方案

第一部分：通用專案設定 (剛好學, 白板, 簡報)

1-3

建立專案、開啟匿名登入、建立資料庫

請依照 **「建立專案」、「開啟匿名登入 (Authentication)」** 和 **「建立資料庫 (Firestore)」** 的步驟操作。

專案名稱建議： Universal-Tools
通用專案規則
4

設定 100 年有效期的安全性規則 (通用工具)

這是給 **剛好學、剛好白板、剛好簡報** 專用的規則，可讓資料在 2125 年前都能讀寫，無需每年更改。 

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 匹配資料庫中的任何文件路徑
    match /{document=**} {
      // 允許讀取和寫入，條件是：當前時間小於 2125 年 1 月 1 日
      allow read, write: if request.time < timestamp.date(2125, 1, 1);
    }
  }
}

第二部分：剛好測專案設定 (必需單獨申請)

A

建立第二個專案（剛好測）

**重要：** 由於剛好測的資料（題庫、成績）需永久保存，我們必須將它與其他工具分開。

請重複步驟 1-3：**「建立專案」、「開啟匿名登入」** 和 **「建立資料庫」**。

專案名稱建議： Quiz-Database
剛好測專用規則
B

設定剛好測的安全性規則

這是專為 **剛好測** 的資料結構設計的規則，確保老師可管理題庫、學生可匿名作答與讀取成績。 

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 匹配資料庫中的任何文件路徑
    match /{document=**} {
      // 允許讀取和寫入，條件是：當前時間小於 2125 年 1 月 1 日
      // (所有資料都設為 100 年效期，無需擔心過期)
      allow read, write: if request.time < timestamp.date(2125, 1, 1);
    }
  }
}

**提示：** 雖然剛好測有特殊的資料結構，但由於我們讓所有使用者都能讀寫，所以可以直接沿用與通用專案相同的 100 年通用規則，**設定方式完全相同**。請同樣貼上 **步驟 4** 的代碼即可。

第三部分：取得設定碼與應用

5

取得設定碼 (Config)

從 **通用專案** 和 **剛好測專案** 分別執行 Web 應用程式註冊（步驟 5），並複製大括號 { 到 } 中間的內容。

您需要複製的內容範例：

apiKey: "AIzaSyD...",
authDomain: "your-project.firebaseapp.com",
projectId: "your-project",
storageBucket: "your-project.appspot.com",
messagingSenderId: "123456...",
appId: "1:123456..."

應用到對應的程式中

根據程式的性質，貼上對應的設定碼：

  • 剛好測（aiquiztools）：
    貼上 **剛好測專案** 的設定碼。
  • 剛好學、剛好白板、剛好簡報：
    貼上 **通用專案** 的設定碼。

點擊各程式右上角的 **「Firebase 設定」** 或 **「設定 (齒輪)」** 按鈕即可貼上並套用。
